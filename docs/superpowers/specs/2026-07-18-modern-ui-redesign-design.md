# Token Studio 现代化 UI 重构设计

日期：2026-07-18
分支基础：`codex/modern-dashboard`（复用其五视图框架与数据逻辑）
范围：看板五个视图 + `/review` 复盘页的视觉与信息架构重构

## 背景与目标

现有 main 版是单页长报表，用户反馈"页面太空、数据铺排不平整、注意力不集中、不够现代"。Codex 分支已拆出五视图，但概览页右侧「运行摘要」栏堆砌杂项，把"每个工具/每个模型用了多少"这一产品初心挤到了二级页面。

**设计验收标准：打开首页 3 秒内能回答——我总共用了多少 Token、花了多少钱、每个工具各占多少、额度还剩多少。**

## 已确认的决策

| 决策点 | 结论 |
| --- | --- |
| 代码基础 | `codex/modern-dashboard` 分支，保留五视图框架 |
| 概览布局 | 方案 A：KPI 横带 + 按工具堆叠趋势 + 工具卡片网格 |
| 导航 | 左侧固定边栏（替换顶部标签） |
| 视觉风格 | 现代数据产品风（Linear / Vercel 式克制精致） |
| 默认主题 | 亮色默认，暗色一键切换 |
| 范围 | 看板五视图 + 复盘页一起换新视觉 |

## 1. 信息架构与导航

- 左侧固定边栏约 220px：品牌区；五个视图入口——概览、活跃度、用量拆解、会话、额度；底部——复盘入口（跳 `/review`）、主题切换。
- 页头右上保留：最后同步时间、采集按钮（含进度轮询）、刷新。
- 响应式：<1100px 侧栏收成图标窄栏；手机端顶栏 + 抽屉菜单。
- 内容区最大宽度 1440px 居中。

## 2. 概览页（核心重做）

删除 Codex 版整个「运行摘要」右栏（今日 Token、峰值小时、本月费用预测、7 天用量、需要关注模块全部移除，不迁移到其他页面）。

三层结构：

1. **KPI 横带（5 卡）**：总 Token、估算费用、缓存命中率（各带 vs 上一周期环比箭头）+ Claude 额度、Codex 额度（进度条 + 重置时间）。未连接/不可读的额度卡自动隐藏；一个都没有时该位置由前三张 KPI 拉伸填满。
2. **主趋势图**：每日用量柱状图按工具堆叠着色（工具颜色全站固定），Token/费用切换，保留 ECharts 缩放滑轨，点图例聚焦单工具。
3. **工具卡片网格**：每个有数据的来源一张卡——logo、名称、Token 量、费用、占比条、Top 模型。点击卡片跳转「用量拆解」并自动应用该来源筛选。来源多于 6 个时聚合尾部为「其他」卡。

概览页保留视图级时间筛选（今天 / 7 天 / 14 天 / 30 天 / 90 天 / 全部 + 自定义区间）。

## 3. 其余四个视图（重排版，不重造）

- **活跃度**：28×24 小时热力图 + 活跃摘要（活跃天数、当前/最长连续、峰值日期、星期分布、工作日/周末占比），沿用现有左右结构，套新视觉。
- **用量拆解**：来源 / 模型 / 项目三维度切换 + Token/费用切换；新增承接从概览移出的 **Token 构成**（Input / Output / Cache / Reasoning 条形拆解）。支持从概览工具卡片带入来源筛选。
- **会话**：真实事件级会话表 + 搜索，数据逻辑不变。
- **额度**：Claude/Codex 窗口详情、使用率、重置时间快照，数据逻辑不变。

## 4. 复盘页

章节内容与结构不动，仅替换视觉皮肤：新字体层级、颜色 token、卡片样式。保持阅读/打印友好，打印始终按亮色输出。

## 5. 视觉系统

### 主题 token（CSS 变量，`data-theme` 挂根节点，localStorage 持久化）

| Token | 亮色（默认） | 暗色 |
| --- | --- | --- |
| 页面底色 | `#f7f7f8` | `#101014` |
| 卡片面 | `#ffffff` | `#1b1b21` |
| 边框 | `#e7e7ea` | `#26262c` |
| 主文字 | `#18181b` | `#ededf0` |
| 次要文字 | `#83838c` | `#8f8f9a` |
| 强调色 | `#5865f2` | `#6673ff` |

### 排版与质感

- 字体：Inter + PingFang SC 系统栈；所有数字 `tabular-nums`；大数字加粗、收紧字距。
- 9px 圆角、1px 边框、极轻阴影、8px 间距网格。不使用玻璃拟态、大面积渐变。

### 图表配色

每个工具全站固定专属色（Claude Code 靛蓝、Codex CLI 紫、OpenCode 青绿、Gemini 蓝、Hermes 橙、OpenClaw 红），五个视图与复盘页统一。ECharts 主题从 CSS 变量取色，切主题时图表同步换色。

## 6. 技术实现

- 栈不变：React + ECharts + Vite；改造 `src/client/dashboard/` 与 `src/client/review/`。
- `App.jsx`（1090 行）拆分：`views/Overview.jsx`、`views/Activity.jsx`、`views/Usage.jsx`、`views/Sessions.jsx`、`views/Quota.jsx` + 共享布局组件（侧栏、页头、KPI 卡、面板）。
- `styles.css` 重写为设计 token 体系，两套主题共用一份布局规则。
- 后端 API、SQLite、采集逻辑零改动。
- 现有 22 个自动化测试保持通过；完成后跑 `npm run build` 生产构建验证。

## 明确不做

- 不新增预测/burn-rate 类衍生指标（额度页现状保留即可）。
- 不做 3D 热力图、排行榜、多套彩色主题。
- 不改数据库 schema 与采集协议。
